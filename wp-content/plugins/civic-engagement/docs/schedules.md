# Schedule Module

## Purpose

Public/private schedule and calendar management.

Separate from event registration.

---

# Current Simplifications

* No recurring schedules.
* No workflow automation.
* No advanced lifecycle management.

---

# Schedule Types

* meeting
* motion
* question
* rep follow-up
* public announcement
* other

---

# Admin Flow

Admin can:

* create schedules,
* set public/private visibility,
* add comments,
* update status,
* archive automatically,
* create schedule from rep/thread.

---

# Public Visibility Rules

* Public schedules become visible from start date.
* Expired schedules move to archive automatically.
* Private schedules remain admin-only.

---

# Suggested Fields

* title
* details
* type
* status
* review date
* internal comment
* response
* start date
* end date

---

# Important Rules

* Create-from-rep/thread only prefills fields.
* Admin must manually review/edit before save.

---

# Future Possibilities

* recurring schedules,
* reminders,
* advanced workflows,
* team assignments.
